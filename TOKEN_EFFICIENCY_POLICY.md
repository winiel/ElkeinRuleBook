## 💡 토큰 절약 기법

### 1️⃣ **컨텍스트 관리**

#### **불필요한 파일 읽기 제한**
```markdown
❌ 나쁜 예: 매번 모든 메모리 파일 읽기
✅ 좋은 예: 필요할 때만 특정 날짜 파일 읽기
```

#### **Memory Compaction 활용**
- 긴 대화는 자동으로 요약됨
- 중요한 정보는 MEMORY.md에 보관
- 일시적인 정보는 memory/YYYY-MM-DD.md에만

#### **중복 정보 제거**
- 같은 설명 반복 금지
- 이전 대화 참조로 대체
- "앞서 말씀드린 대로..." 활용

### 2️⃣ **출력 최적화**

#### **간결한 응답**
```markdown
❌ 나쁜 예:
"네, 알겠습니다! 지금 바로 해당 작업을 진행하도록 하겠습니다. 
먼저 첫 번째로 A를 하고, 그 다음에 B를 하고, 마지막으로 C를 할 것입니다.
이렇게 하면 원하시는 결과를 얻으실 수 있을 것으로 생각됩니다."

✅ 좋은 예:
"알겠습니다! A → B → C 순서로 진행하겠습니다."
```

#### **불필요한 마크다운 제거**
```markdown
❌ 나쁜 예:
## 결과
### 성능
#### FPS
- 14.2 FPS
- 목표 15 FPS

✅ 좋은 예:
**결과**: 14.2 FPS (목표 15)
```

#### **핵심만 전달**
- 기술적 세부사항은 필요할 때만
- 사용자가 물어본 것에 집중
- 부가 설명은 최소화

### 3️⃣ **Tool Call 최적화**

#### **Batch Tool Calls**
```python
❌ 나쁜 예: 순차적 호출
exec("command1")
exec("command2")
exec("command3")

✅ 좋은 예: 한 번에 실행
exec("command1 && command2 && command3")
```

#### **결과 재사용**
- 같은 파일을 여러 번 읽지 않기
- 캐시 가능한 정보는 기억하기
- 불필요한 재확인 제거

#### **판단 가이드: 새 에이전트 vs 기존 에이전트**

**새 에이전트 배포:**
- ✅ 완전히 다른 작업
- ✅ 독립적인 프로젝트
- ✅ 이전 작업 완료됨
- 예: "Hailo 최적화" 완료 → "동영상 6개 처리" 시작

**기존 에이전트에게 전달 (sessions_send):**
- ✅ 같은 작업의 수정
- ✅ 추가 요구사항
- ✅ 삭제/변경 요청
- ✅ 에이전트 아직 작업 중
- 예: "15 FPS" → "16 FPS로 변경"
